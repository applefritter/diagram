Reverse Proxy

Another options for handling more requests is to use a reverse-proxy server

We introduce the reverse-proxy before our application-server. Now requests to our website go directly to the reverse-proxy not our app-server.

The reverse-proxy runs its own web-server software but it is configured to route requests to other servers.

This implies we can add multiple app-servers and have our reverse-proxy distribute requests to them.

This is called scaling horizontally...
